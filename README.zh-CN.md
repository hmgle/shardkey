# ShardKey

[English](README.md)

ShardKey 是一个纯前端、离线运行的秘密分享工具，目前提供两种模式：

- `经典问答`：设置问题、答案和门限值，答对足够多问题后解锁秘密。
- `多人合力恢复`：把一个秘密拆成 `N` 个分片链接/JSON，集齐至少 `K` 个分片后恢复秘密。

它更适合被理解为一种可分享的离线挑战格式，而不是高强度的秘密共享系统。

## 快速开始

直接在浏览器中打开 `index.html` 即可使用：

```bash
open index.html
```

也可以起一个本地静态服务：

```bash
python3 -m http.server 8000
# 然后访问 http://localhost:8000
```

## 核心功能

1. 创建挑战：输入秘密、设置问题与答案、配置门限值
2. 分享挑战：复制更紧凑的挑战链接，或下载 JSON 文件
3. 求解挑战：通过链接或 JSON 导入，回答问题后尝试解锁

### 多人合力恢复模式

1. 切换到 `多人合力恢复` 模式。
2. 输入秘密、总分片数 `N` 和恢复门限 `K`。
3. 生成分片链接后，把不同分片分别发给不同的人或保存到不同位置。
4. 恢复端打开任意一个分片链接，或导入分片 JSON，然后继续粘贴其他分片，直到至少收集到 `K` 个。
5. 在浏览器本地完成秘密恢复。

### 多答案（同一问题允许多个正确答案）

创建挑战时，每个问题都可以添加多个“备选答案”：点击“+ 添加备选答案”增加输入框即可。任一备选答案正确就算答对该题（大小写不敏感）。

答案文本可以包含任意字符（包括 `|`），不再依赖分隔符来区分多个答案。

## 技术特性

- 纯 HTML/CSS/JavaScript，零外部依赖
- 使用 Web Crypto API 的 AES-256-GCM 加密共享秘密
- 使用 GF(256) 上的 Shamir Secret Sharing 拆分随机内容密钥
- 使用 PBKDF2（每题独立 salt）增强答案抗爆破
- 在 `经典问答` 模式下，每个正确答案都会解开一个加密后的 Shamir share
- 分享链接使用紧凑打包格式，以尽量缩短 URL，方便转发和传播
- 所有计算都在浏览器本地完成，支持 `file://` 直接打开
- 在浏览器支持时，优先使用 Web Worker 将生成和求解放到后台线程

## 安全说明

- ShardKey 的定位是“离线问答解锁器”，不是经过审计的密码学门限秘密共享系统替代品。
- 分享数据不包含明文秘密、正确答案或答案哈希
- 它的门限逻辑用于控制离线解锁，不适合替代密码库等高价值秘密存储
- 只有在收集到足够 share 并通过 AEAD 认证后，秘密才会被恢复

### 重要限制

- 弱答案仍可能被离线字典攻击；PBKDF2 只能显著减慢爆破速度，并不能彻底阻止。
- 同一题允许多个正确答案会降低“猜中任一变体即可通过”的难度；尽量避免把常见同义词/近义词全部加入答案列表。
- 为了稳健性，应用设置了实用的输入上限（当前：秘密 ≤ 1024 字节，题目 ≤ 64，门限 ≤ 64）。更大的挑战建议拆分。
- 如果你需要更强的保密性，建议使用高熵秘密（例如在秘密后追加随机串），并优先选择更高的门限值。
- `经典问答` 模式仍然依赖答案强度；弱答案可能让攻击者离线恢复单题的加密 share。
- 如果你需要真正高强度的密码学保护，请改用经过审计的秘密共享系统或密码管理器，而不是仅依赖挑战问答。
