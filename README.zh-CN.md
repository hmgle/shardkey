# ShardKey

[English](README.md)

ShardKey 是一个纯前端的“答题解锁秘密”工具。
你设置问题、答案和门限值（至少答对多少题），系统生成可分享的挑战链接或 JSON 文件。解题者只有答对足够多问题，才能恢复秘密内容。

## 快速开始

直接在浏览器中打开 `index.html` 即可使用：

```bash
open index.html
```

也可以起一个本地静态服务：

```bash
python3 -m http.server 8000
# 然后访问 http://localhost:8000
```

## 核心功能

1. 创建挑战：输入秘密、设置问题与答案、配置门限值
2. 分享挑战：复制带挑战数据的链接，或下载 JSON 文件
3. 求解挑战：通过链接或 JSON 导入，回答问题后尝试解锁

### 多答案（同一问题允许多个正确答案）

创建挑战时，每个问题都可以添加多个“备选答案”：点击“+ 添加备选答案”增加输入框即可。任一备选答案正确就算答对该题（大小写不敏感）。

答案文本可以包含任意字符（包括 `|`），不再依赖分隔符来区分多个答案。

## 技术特性

- 纯 HTML/CSS/JavaScript，零外部依赖
- 使用 BigInt 进行大整数运算
- 使用 Web Crypto API 进行 SHA-256 哈希
- 使用 PBKDF2（每题独立 salt）增强答案抗爆破
- 所有计算都在浏览器本地完成，支持 `file://` 直接打开

## 安全说明

- 分享数据不包含明文秘密、正确答案或答案哈希
- 只有足够数量的正确答案组合才能通过门限恢复秘密
- 恢复出的秘密会通过内嵌校验字段验证，防止错误答案误解锁

### 重要限制

- 弱答案仍可能被离线字典攻击；PBKDF2 只能显著减慢爆破速度，并不能彻底阻止。
- 同一题允许多个正确答案会降低“猜中任一变体即可通过”的难度；尽量避免把常见同义词/近义词全部加入答案列表。
- 如果你需要更强的保密性，建议使用高熵秘密（例如在秘密后追加随机串），并优先选择更高的门限值。
- Mignotte/CRT 不是 Shamir 门限方案，也不具备“完美保密”；不要把它当作经过严格审计的密码学秘密共享替代品。
